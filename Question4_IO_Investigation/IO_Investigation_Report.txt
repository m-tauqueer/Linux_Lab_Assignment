File Descriptor Observations:
The ls -l /proc/$fish_pid/fd command showed the current shell's open file descriptors:
- fd 0 (stdin), fd 1 (stdout), fd 2 (stderr) all point to /dev/pts/1 (the terminal)
- fd 10 points to anon_inode:inotify

This shows that every process tracks its open files as numbered descriptors in /proc.

How Linux Manages File I/O:
Linux represents open files as file descriptors (integers 0, 1, 2 for stdin/stdout/stderr). Each descriptor in a process points to an entry in the kernel's open file table. The /proc filesystem exposes these as symlinks under /proc/PID/fd, allowing inspection without special tools. Redirection operators (> and 2>) replace or duplicate descriptors so output goes to files instead of the terminal. Resource limits in /proc/self/limits cap how many files a process can open simultaneously.
